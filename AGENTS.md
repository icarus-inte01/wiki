# Wiki 규칙

이 문서는 `~/doc/wiki` 작업 시 AI(Sisyphus)가 항상 따라야 할 규칙을 정의합니다.

## index.md 관리 규칙

> **새 `.md` 파일이 추가되거나 삭제될 때마다, 해당 디렉터리의 `index.md`에 제목과 링크를 반영해야 합니다.**

### 추가 규칙

1. `docs/` 아래 특정 디렉터리에 새 `.md` 파일(예: `docs/dev/mcp.md`)이 생성되면, 같은 디렉터리의 `index.md`(예: `docs/dev/index.md`)에 항목을 **반드시 추가**해야 합니다.

2. 추가 형식:
   ```markdown
   - **[문서 제목](파일명.md)** — 한 줄 설명
   ```

3. 문서의 제목은 해당 `.md` 파일의 첫 번째 `# ` 헤딩을 읽어서 사용합니다.

4. 카테고리(`## 섹션명`)가 적절하면 해당 섹션 아래에 배치하고, 적절한 섹션이 없으면 새 섹션을 추가합니다.

5. 이미 index.md에 등록된 파일은 중복 추가하지 않습니다.

### 삭제 규칙

1. `docs/` 아래 특정 디렉터리에서 `.md` 파일이 삭제되면, 같은 디렉터리의 `index.md`에서 **해당 파일의 링크 항목을 반드시 제거**해야 합니다.

2. 항목 제거 시, 해당 항목만 제거하고 섹션 헤더나 다른 항목은 유지합니다. 만약 항목 제거 후 섹션이 비어 있게 된다면 빈 섹션도 함께 제거합니다.

3. 제거 대상 식별: `index.md`에서 `(파일명.md)` 패턴으로 링크된 파일을 찾아 제거합니다.

### 이동 규칙

1. `docs/` 아래 특정 디렉터리에서 다른 디렉터리로 `.md` 파일이 이동되면, **두 index.md를 모두 갱신**해야 합니다.
   - **원본 디렉터리**의 `index.md` → 해당 파일 링크 항목 **제거**
   - **대상 디렉터리**의 `index.md` → 해당 파일 링크 항목 **추가**

2. 제거 방식은 삭제 규칙을 따르고, 추가 방식은 추가 규칙을 따릅니다.

3. 파일명이 변경되지 않았다면 기존 제목(`# ` 헤딩)을 재사용하고, 변경되었다면 새 파일의 `# ` 헤딩을 읽어서 사용합니다.

### 예외

- `index.md` 자체는 목록에서 제외합니다.
- 외부 참조용 문서 등 index.md에 노출할 필요가 없는 파일은 명시적으로 제외할 수 있습니다.

### 적용 예시

**추가**: `docs/dev/mcp.md` 생성 → `# MCP (Model Context Protocol)` 읽음 → `docs/dev/index.md`에 추가:

```markdown
- **[MCP (Model Context Protocol)](mcp.md)** — AI Application과 외부 도구 연결을 위한 표준 프로토콜
```

**삭제**: `docs/dev/old-tool.md` 삭제 → `docs/dev/index.md`에서 `- **[Old Tool](old-tool.md)** — ...` 항목 제거
